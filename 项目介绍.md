# 桌面级温湿度检测仪器 — 项目技术文档

> 基于 STM32G030K6T6 的低功耗桌面温湿度监测仪，支持实时环境温湿度显示与供电电压监测。

---

## 1. 项目概述

### 1.1 项目简介

本项目旨在设计并实现一款 **桌面级温湿度检测仪器**。该仪器以 **STM32G030K6T6** 单片机为核心，搭载 SHT40 高精度数字温湿度传感器，能够 **实时监测并显示环境的温湿度状况**。设备采用 **低功耗设计**，显示完毕后自动进入休眠模式，通过按键触发唤醒，兼顾了交互便利性与长期运行的能耗需求，适用于家庭、办公室、宿舍等多种室内场景。

### 1.2 设计目标

| 目标 | 实现方式 |
|------|----------|
| 高精度温湿度实时采集 | SHT40 传感器 通过软件模拟 I2C 通信（GPIO 开漏 + `delay_us(5)`，约 100kHz） |
| 直观的数据显示 | 3 位段码数码管，SN74HC595 级联驱动动态扫描 |
| 低功耗运行 | 显示完成后自动进入 WFI 睡眠模式，按键 EXTI 唤醒 |
| 按需交互 | 单击显示温湿度，双击显示电池/供电电压 |
| 高可移植性 | 软件 I2C 替代硬件 I2C 外设，无引脚锁定限制 |

### 1.3 适用场景

- **家庭**：卧室、客厅温湿度监测
- **办公室**：工位桌面环境感知
- **宿舍**：便携式室内气候监测
- **小型实验室**：低成本环境参数记录

## 4. 核心功能说明

### 4.1 软件模拟 I2C 协议

传统 STM32 硬件 I2C 存在兼容性问题和引脚锁定限制。本项目采用 **GPIO 软件模拟 I2C 时序**，实现更灵活、可移植的通信方案。

| 要素 | 实现 |
|------|------|
| 底层延时 | `delay_us(5)` — 基于 SysTick 硬件计数器的微秒延时，约 100kHz |
| GPIO 模式 | 开漏输出（`GPIO_MODE_OUTPUT_OD`）+ 内部上拉 |
| 起始/停止 | SCL 高电平时 SDA 跳变（标准 I2C Start/Stop 时序） |
| 数据收发 | `I2C_SendByte()` / `I2C_ReceiveByte()` 逐位操作 |
| ACK 检测 | `I2C_WaitAck()` 在第 9 个时钟读取 SDA 状态 |
| CRC/校验 | 由 SHT40 传感器内部完成，固件仅传原始数据 |

**I2C 通信流程（SHT40 读数）：**

```
1. I2C_Start() → 发送 0x44<<1 (写地址) → 检查 ACK
2. 发送命令 0xFD (高精度温湿度测量) → 检查 ACK
3. I2C_Stop()
4. HAL_Delay(10ms) 等待传感器测量完成
5. I2C_Start() → 发送 (0x44<<1)|0x01 (读地址) → 检查 ACK
6. 连续读取 6 字节数据 (5 次 ACK + 1 次 NACK)
7. I2C_Stop()
8. 公式转换：t = 175 * St / 65535 - 45,  RH = 125 * Sr / 65535 - 6
```

### 4.2 段码数码管驱动流程

数码管通过 3 片 SN74HC595 移位寄存器级联控制：

| 编号 | 作用 | 控制内容 |
|------|------|----------|
| SN_LED1 | 第一排段选 | a-g + 小数点 8 段 |
| SN_LED2 | 第二排段选 | a-g + 小数点 8 段 |
| SN_DIG | 6 位位选 | 低电平有效，选中对应位 |

**`ShowNum(row, column, value)` 显示流程：**

1. 根据 `row`（第 1/2 排）选择 SN_LED1 或 SN_LED2
2. 根据 `column`（第 1/2/3 位）将 SN_DIG 对应位置低、其余置高
3. `SN74HC595_Send_Data()` 逐位串行输出数据，产生 SCLK/RCLK 锁存
4. `SysCtlDelay(350)`（约 50μs）位选消影，防止残影
5. 段选送 `0x00` 消影，防止错位

**共阴极段码表（0~9 + 小数点）：**

| 数字 | 段码 (a-g+dp) | 实际值 |
|------|---------------|--------|
| 0 | 00111111 | 0x3F |
| 1 | 00000110 | 0x06 |
| 2 | 01011011 | 0x5B |
| 3 | 01001111 | 0x4F |
| 4 | 01100110 | 0x66 |
| 5 | 01101101 | 0x6D |
| 6 | 01111101 | 0x7D |
| 7 | 00000111 | 0x07 |
| 8 | 01111111 | 0x7F |
| 9 | 01101111 | 0x6F |
| DP | 10000000 | 0x80 |

### 4.3 显示功能与工作模式

| 模式 | `displayMode` | 触发方式 | 显示内容 |
|------|:---:|------|------|
| 温湿度模式 | 0 | 单击按键 | 温度（3 位）→ 湿度（3 位），约 2.4 秒后休眠 |
| 电压模式 | 1 | 双击按键 | 电压值（3 位），约 1.2 秒后休眠 |

**显示切换机制（TIM14 定时器中断）：**

- TIM14 配置：`Prescaler = 63`, `Period = 29`，中断频率 ≈ 16.67kHz（~60μs/次）
- `updata_flag` 在每次中断中递增，通过计数实现时间分片：
  - `updata_flag ≤ 5000`：显示温度/电压（约 300ms）
  - `5000 < updata_flag ≤ 10000`：显示湿度（约 300ms，仅温湿度模式）
  - `updata_flag > 10000`：重置计数，`sleep_flag++`
- `sleep_flag ≥ 2`：关闭数码管，设置 `sleepStatus = 1`，进入 WFI 休眠

**单击 / 双击检测逻辑：**

```
下降沿中断 (PB5) → 设置 KeyStatus = KEY_SHAKE_STATE
    → main 消抖 10ms
    → 第一次按键 → 启动 500ms 双击窗口 (clickPending=1)
    → 窗口内第二次按键 → 双击 → displayMode=1 (电压模式)
    → 窗口超时 → 单击 → displayMode=0 (温湿度模式)
```

### 4.4 ADC 电压采集

| 项目 | 配置 |
|------|------|
| ADC 外设 | ADC1, 通道 9 (PB1) |
| 分辨率 | 12 位 |
| 采样时间 | 160.5 ADC 时钟周期 |
| 时钟分频 | PCLK/2 |
| 转换方式 | 软件触发，单次转换 |
| 输入电压范围 | 0 ~ 3.3V（实际分压 ~1.65V） |
| 计算公式 | `Vol_Value = (ADC_Value × 3.3 / 4095) × 100 × 2`（厘伏单位） |

### 4.5 低功耗休眠

| 阶段 | 操作 |
|------|------|
| 进入休眠 | `HAL_SuspendTick()` 暂停 SysTick → `HAL_PWR_EnterSLEEPMode(WFI)` |
| 唤醒源 | PA5 按键下降沿外部中断 |
| 唤醒后 | `HAL_ResumeTick()` 恢复 SysTick，`sleepStatus = 0` |

## 6. 编译与部署步骤

### 6.1 编译流程

1. **打开工程**：启动 STM32CubeIDE，选择 `File → Open Projects from File System...`，导入 `prj_G030K6_3_Seg_Soft_I2C` 目录。

2. **确认编译配置**：
   - 右键工程 → `Properties → C/C++ Build → Settings`
   - 确认 MCU 为 `STM32G030K6Tx`
   - 确认链接脚本为 `STM32G030K6TX_FLASH.ld`

3. **编译**：
   - 菜单栏 `Project → Build All`（快捷键 `Ctrl+B`）
   - 或右键工程 → `Build Project`
   - 编译输出在 `Debug/` 目录：`prj_G030K6_3_Seg_Soft_I2C.elf`、`.hex`、`.bin`

### 6.2 烧录部署

1. **硬件连接**：通过 ST-LINK 连接 STM32G030K6T6 开发板（SWDIO + SWCLK + GND + 3.3V）。

2. **烧录**：
   - 右键工程 → `Run As → STM32 Cortex-M C/C++ Application`
   - 或工具栏 `Run → Debug Configurations...` 配置 ST-LINK 调试器
   - 点击 `Run` 按钮下载固件

3. **验证**：烧录完成后，按下 PB5 按键，数码管应亮起并显示温度/湿度值。

### 6.3 命令行编译（可选）

```bash
cd prj_G030K6_3_Seg_Soft_I2C/Debug
make -j4 all
```

```c
struct DEVICE_PARAMTER {
    volatile uint8_t KeyStatus;        // 按键状态 (0/1)

```
